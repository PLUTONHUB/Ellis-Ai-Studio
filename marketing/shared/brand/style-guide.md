# Writing Style Guide

- Lead with the customer problem, outcome, or useful lesson.
- Prefer short sentences and concrete verbs.
- Explain technical work in plain language before naming tools.
- Use one main idea and one clear call to action per post.
- Break social copy into short, scannable paragraphs.
- Use approved links, names, results, and visual assets only.

Before publishing, check accuracy, clarity, accessibility, customer privacy, and whether the claim is supported by the source material.
