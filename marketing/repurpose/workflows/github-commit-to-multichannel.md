# GitHub Commit to Multichannel Workflow

1. Select a commit with a meaningful audience benefit and verify the facts.
2. Use the shared storytelling framework to create a long-form source narrative.
3. Generate channel drafts using each platform module.
4. Create website changelog, portfolio, newsletter, and blog candidates only when approved source material supports them.
5. Save each output in the content library, route it through scheduling, and record performance for future reuse.
