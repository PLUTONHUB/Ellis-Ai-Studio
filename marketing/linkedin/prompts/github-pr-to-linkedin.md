# Generate a LinkedIn Post from a GitHub Pull Request

Use this prompt after a pull request is merged or approved for announcement.

```text
Create a LinkedIn post from the pull request details below.

Audience: [target audience]
Post type: [Project Launch | Feature Update | Case Study | Founder Update | Milestone Announcement]
Voice: clear, credible, conversational, and concise
Goal: translate technical work into a customer-relevant story

PR title and description:
[paste PR title and description]

Key changes and review context:
[paste summary]

Approved context, metrics, links, or claims:
[paste context]

Constraints:
- Do not invent facts, metrics, customer names, or timelines.
- Exclude secrets, internal-only details, and unverified claims.
- Lead with the problem or outcome, not a list of implementation details.
- Use a strong hook, short paragraphs, 3–5 relevant hashtags, and a call to action.
- Return the finished post only, followed by a one-line fact-check list.
```
