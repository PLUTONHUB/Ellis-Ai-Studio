# Campaign System

Campaigns organize a related set of content around one audience outcome. They consume `marketing/shared/` and reference content-library IDs rather than duplicating platform copy.

## Campaign types

- Product launches
- Demo launches
- Client launches
- Website releases
- Feature releases
- Seasonal campaigns

## Lifecycle

- `upcoming/` — Validated briefs and planned assets.
- `active/` — In-market campaigns with scheduled content and monitoring notes.
- `completed/` — Final results, lessons, and reusable examples.
- `templates/` — Reusable briefs and checklists.

Every brief should define the audience, goal, shared message, approved proof, channels, CTA, owner, dates, measurement plan, and asset dependencies.
