# Content Classification

Classify each verified event before draft generation. Classification is a recommendation that a reviewer can change.

| Classification | Common evidence |
| --- | --- |
| Feature Update | Shipped capability or meaningful user workflow change |
| Website Launch | Production website or major release is live |
| Client Launch | Client-approved launch or public outcome |
| Bug Fix | Customer-relevant issue resolved; avoid routine maintenance |
| Design Improvement | Clear customer-experience or conversion improvement |
| New Service | Approved service offer or delivery capability |
| Case Study | Permission-safe story with verified context and result |
| Portfolio Update | Approved project, demo, or design artifact update |

When an event fits multiple classifications, generate one primary recommendation and record alternatives for review. Never infer client permission, metrics, or launch status.
