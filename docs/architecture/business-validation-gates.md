# Business Validation Gates

Evaluate every new epic or major feature before significant implementation. Record the answers in the epic, its ADR, or both. A feature must pass the gates below before it is prioritized.

## Gate 1 — Revenue

Does it directly help acquire clients or increase revenue?

If no, do not prioritize it.

## Gate 2 — Client Value

Does it improve the client experience or a measurable business outcome?

If no, re-evaluate the feature.

## Gate 3 — Automation

Does it eliminate manual work for Ellis AI Studio or its clients?

If no, document why the feature remains valuable.

## Gate 4 — Reusability

Can the capability be reused across multiple clients or internal systems?

If no, explain why a one-off implementation is justified.

## Gate 5 — Strategic Alignment

Does it align with Ellis AI Studio's mission of building AI-powered growth systems?

If no, move it to the roadmap.

## Decision record

Use this concise record with each major feature:

| Gate | Answer | Evidence / rationale | Outcome |
| --- | --- | --- | --- |
| Revenue | Yes / No |  | Pass / Do not prioritize |
| Client value | Yes / No |  | Pass / Re-evaluate |
| Automation | Yes / No |  | Pass / Document exception |
| Reusability | Yes / No |  | Pass / Document exception |
| Strategic alignment | Yes / No |  | Pass / Move to roadmap |

**Overall decision:** Approved / Deferred / Rejected
